# ModuFlow Opportunities

## Multi-Project Team Adoption

- User: people managing multiple product or implementation projects.
- Problem: existing project structures, decisions, reports, benchmarks, and status updates are scattered.
- Desired outcome: install ModuFlow once, adopt it safely per project, and get project-local plus portfolio-level visibility.
- Evidence: user conversation on 2026-06-11.
- Risk: forcing a canonical structure too early could disrupt existing projects.
- Recommendation: start with migration and project profile work before portfolio dashboards.
