# ModuFlow Implementation Plan

## Goal

Create a lightweight, update-friendly PM execution plugin that combines Claude Cowork Productivity/Product Management patterns, GitHub Spec Kit structure, Superpowers execution practices, and Codex Product Design/Data/Documents capabilities.

## Location

`/Users/dongwon.lee/workhub/company/shared/context-tools/moduflow`

## Architecture

- `Git` is the source of truth.
- `Productivity` is used as a dashboard/view pattern, not the state store.
- `Product Management` contributes PM artifact patterns.
- `Spec Kit` contributes spec/plan/task structure.
- `Superpowers` contributes execution, subagent, verification, and review process.
- `Codex Product Design`, `Data Analytics`, `Documents`, and `Presentations` are bridged as optional adapters.

## Artifact Model

```text
Inbox -> Opportunity -> Git Issue -> Spec/PRD -> Analysis/Metrics
      -> Design/Prototype -> Plan/Tasks -> Execution -> PR
      -> Verification -> Release -> Stakeholder Update
```

The roadmap is a priority and timing view over opportunities, issues, specs, and releases. It is not the source of truth.

## Build Steps

1. Scaffold plugin manifest and README.
2. Add `/product:*` command docs with next-step guidance.
3. Add bridge skills for PM routing, Git artifact model, roadmap, progress dashboard, source adapters, Superpowers execution, data analysis, and design/prototype work.
4. Add worker role definitions for parallel subagent execution.
5. Add templates for issues, specs, status, roadmap, and updates.
6. Add vendor lock and adapter docs so upstream sources can be updated or swapped.
7. Add a local validator that checks required files exist.
