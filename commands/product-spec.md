# /product:spec

Turn a Git issue into a spec.

## Do

1. Create `specs/<issue-id>-<slug>/spec.md`.
2. Define problem, users, goals, non-goals, requirements, acceptance criteria, risks, and open questions.
3. Use Spec Kit structure where useful.
4. Keep the Git issue linked.

## Next

- `/product:analyze` if metrics or evidence are needed
- `/product:design` if UX flow is unclear
- `/product:plan` when the spec is ready
