# No-Issue Declarations

> Each bare (non-heading, non-blockquote, non-backtick) line below is one declaration that a specific behavior change ships without an issue. A declaration is valid only when `git blame` attributes its line to a human identity listed in `.moduflow/humans.json` (or an equivalent human PR approval exists).
> Format: `<date> <scope-or-paths> — <reason>`
> Declarations for a release are listed in that release's `human-review.ko.md` packet. This file should hold zero declarations at most releases — an entry here is the exception, not the norm.
