# 명세: 이중 언어 산출물 뷰 (영문 canonical, 한글은 읽기용)

이슈: `049-bilingual-artifact-view`
이전: `045-issue-graph-visualization` (확장 대상 패널) · `047-issue-artifact-drilldown` (토글이 붙는 패널) · 다음: `product:plan 049`

> 이 파일은 영문 `spec.md`의 **한글 읽기용 sidecar**입니다. canonical(원본·진실)은 영문이며, 이 한글본은 사람이 검토할 때만 패널에서 보여집니다(049).

## 먼저 명확화 (사용자와 확정, 2026-06-28)

1. 무엇이 canonical인가? → **영문.** "영문으로 만드는 게 맞고." 표준이고 외부 호환되며 진실의 원천.
2. 한글은 어디에 나오나? → **사람이 보는 뷰(047 패널)에서만.** "사람이 보는 것만 한글로." canonical 파일에는 넣지 않음.
3. 기존 영문 명세를 소급 번역하나? → **아니오. 신규 산출물부터.** "신규부터 그렇게 하자." 기존 6개 영문 명세는 그대로.
4. 패널에서 기계 번역? → **아니오** — zero-backend. 한글은 작성 시점에 sidecar로 직접 쓰고, 런타임 번역은 안 함.

## 문제

ModuFlow의 기획 산출물(spec/plan/tasks)은 결정에 따라 영문(canonical)으로 작성됩니다. PM이 이슈의 047 패널을 열어 "무엇을 기획했는지" 검토할 때 영문을 읽게 되어 한국어 사용자에게 느립니다. 완전한 이중 언어 저장은 과투자로 기각됐고, 더 싼 형태를 택했습니다: 영문을 canonical로 유지하고, **신규** 산출물 옆에 한글 sidecar를 쓰며, 패널이 필요 시 보여줍니다.

## 목표

1. **sidecar 컨벤션**: 한글 번역을 영문 파일 옆에 `<이름>.ko.md`로 둠 (예: `spec.md` 옆 `spec.ko.md`). 영문은 canonical로 불변.
2. **패널 언어 토글**: 047 패널이 `English / 한글`을 제공. 산출물에 `.ko.md`가 있으면 토글로 한글을 표시하고, 없으면 영문으로 폴백(선택적 — 빈 한글 화면 없음).
3. **신규부터 정책**: `product:spec`/`plan` 워크플로가 신규 산출물 작성 시 `.ko.md` sidecar를 함께 작성. 기존 영문은 소급 번역하지 않음.

## 비목표

- 런타임/기계 번역 (zero-backend 제약).
- 한글을 canonical로 삼거나 canonical 파일을 직접 번역.
- 기존 6개 영문 명세 소급 번역 (신규부터 — 필요 시 산출물별로 나중에 추가 가능).
- 이슈/지식 그래프 UI(이미 045에서 한글) 또는 메모리 본문 번역.

## 제안 해법

1. **수집**: `_collect_issue_artifacts`가 각 산출물에 `.ko.md` 형제 내용을 `ko` 필드로 첨부(있을 때). `*.ko.md`는 별도 산출물로 나열하지 않음. 각 산출물은 `{name, label, md(영), ko(또는 null)}`을 가짐.
2. **패널 토글**: `render_issue_panel`에 최상위 `English / 한글` 스위치 추가. 기본 영문(canonical). 한글로 전환 시 각 산출물을 `ko`(있으면)로, 없으면 영문으로 재렌더(산출물별 폴백). Markdown/Mermaid 렌더 방식은 동일.
3. **정책**: `product-spec.md`(및 046 템플릿 노트)에 신규 산출물은 `.ko.md` sidecar를 함께 쓴다고 문서화. 컨벤션일 뿐 — 번역 누락으로 빌드를 깨지 않음.

## 검토한 대안

- **완전 이중 언어 저장 + drift 게이트** — 기각(과투자; 사용자가 "보는 것만 한글, 신규부터"를 명시적으로 선택).
- **런타임 marked + 번역 API** — 기각: zero-backend 위배, 키 필요, 조회마다 비용.
- **한글을 canonical로** — 기각: 영문이 상호운용 가능한 진실의 원천.
- **한 파일에 영/한 병기** — 기각: canonical 파일을 어지럽히고 길이가 두 배. sidecar가 canonical을 깨끗하게 유지.

## 수용 기준

1. `<이름>.ko.md` 형제가 해당 산출물에 첨부되고, 별도 산출물로 나열되지 않음.
2. 047 패널이 `English / 한글` 토글을 표시; 한글은 sidecar를 렌더하고 없으면 산출물별로 영문 폴백; 기본은 영문.
3. `.ko.md`가 전혀 없으면 패널은 지금과 똑같이 동작(토글·빈 한글 없음).
4. `product-spec.md`가 신규 산출물 sidecar 정책을 문서화(게이트 아님, 컨벤션).
5. `release_check` 통과; 테스트가 sidecar 첨부·토글 렌더·sidecar 없음 폴백을 커버.

## 위험 및 열린 질문

- 위험: 영문이 바뀌면 sidecar가 stale (048 drift 문제의 prose 버전). 완화: 이번엔 게이트 범위 밖, 노트만. 향후 `--drift` 확장이 stale `.ko.md`를 표시할 수 있으나 지금은 아님.
- 열림: 토글 단위 — 전역 스위치 vs 산출물별. 계획: 전역 스위치, 가장 단순.
- 열림: sidecar가 0개면 토글을 완전히 숨길지. 계획: 예(미사용 시 어지럽힘 없음).
